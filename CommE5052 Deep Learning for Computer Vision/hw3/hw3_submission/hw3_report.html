1. Task Overview

The goal of this task is to build an image captioning system using a Vision-Language Model (VLM). Given an RGB image as input, the model should output a natural language caption describing the image content.

We adopt a LLaVA-style architecture combining a pretrained Vision Transformer (ViT) encoder, an MLP projector, and a pretrained Qwen3-based causal language model decoder. To keep computational cost low, we apply Parameter-Efficient Fine-Tuning (PEFT) via LoRA (Low-Rank Adaptation), limiting trainable parameters to under 10M.

Dataset

Training Set

10,000 images from MS-COCO with multiple captions per image. Images are stored as 000000000000.jpg format. Captions are provided in train.json with image_id and caption fields.

Validation Set

2,000 images with ground-truth captions in val.json. Used to evaluate CIDEr and CLIPScore. Each image has multiple reference captions for evaluation.

Vision Encoder

We use CLIP ViT-Base/16 loaded via timm (vit_base_patch16_clip_224.openai). The image is divided into 16×16 patches, yielding 196 patch tokens of dimension 768. The CLS token is discarded; only patch tokens are passed to the projector. The vision encoder is fully frozen during training.

MLP Projector

A 2-layer MLP maps visual tokens from dimension 768 to the language model's hidden dimension 1024: 

Linear(768 → 1024)
GELU activation
Linear(1024 → 1024)
The projector is fully trainable and bridges the visual and language modalities.

Language Decoder

The provided Qwen3-based decoder has 28 transformer layers with hidden size 1024, 16 attention heads (8 KV heads, GQA), head dim 128, and RoPE positional encoding. Visual tokens are prepended to text token embeddings before being fed into the decoder. Only LoRA parameters are trained in this module.

Input Construction

During training, the input sequence is constructed as:

[visual tokens (196)] + [<im_start>] + [caption tokens] + [<im_end>]

Loss is computed only on text tokens using cross-entropy, ignoring padding token id 151643.

3. LoRA Implementation

LoRA (Low-Rank Adaptation) avoids full fine-tuning by injecting trainable low-rank matrices into frozen weight matrices. For a pretrained weight W ∈ ℝd×k, LoRA adds:

W' = W + ΔW = W + B·A
where A ∈ ℝ^(r×k), B ∈ ℝ^(d×r), r << min(d, k)

The output is scaled by α/r, where α is the LoRA alpha hyperparameter. Only A and B are trained; W remains frozen.

Implementation Details

LoRA is applied to q_proj and v_proj in all 28 attention layers of the decoder using loralib. The pretrained weights are copied into the LoRA linear layers, and lora.mark_only_lora_as_trainable() freezes all non-LoRA decoder parameters.

# Replace q_proj with LoRA version
new_q = lora.Linear(in_dim, q_out, r=r, lora_alpha=alpha, bias=False)
new_q.weight = attn.q_proj.weight  # copy pretrained weights
attn.q_proj = new_q

lora.mark_only_lora_as_trainable(self.decoder)

4. Training Setup

Hardware

8× NVIDIA H200 (143 GB each) using PyTorch DistributedDataParallel (DDP) with NCCL backend. Launched via torchrun --nproc_per_node=8.

Optimizer

AdamW with lr=2e-4, weight_decay=0.01. Learning rate schedule: CosineAnnealingLR over 10 epochs. Gradient clipping at norm 1.0.

Data

Batch size 32 per GPU (256 effective). Image augmentation: random horizontal flip, color jitter. Resize to 224×224. Normalized with CLIP mean/std. Max caption length: 64 tokens.

Tokenizer

Custom Qwen3Tokenizer (BPE). Special tokens: <|im_start|> (start), <|im_end|> (end), <|endoftext|> (pad, id=151643). Padding tokens excluded from loss via ignore_index=151643.

7. Observations & Analysis

Effect of LoRA Rank

Comparing Exp 1 (r=8) and Exp 2 (r=4), a lower rank with higher alpha yielded better performance. This suggests that for this task, a higher effective scaling factor (α/r=8 vs 2) is more important than the rank dimension. Exp 3 (r=16, α=16) performed similarly to Exp 2 in CIDEr but slightly lower in CLIPScore, indicating diminishing returns from increasing rank alone.

Effect of LoRA Alpha

Alpha controls the magnitude of the LoRA update via the scale factor α/r. Exp 2 with α=32, r=4 gives a scale of 8.0, the highest among our experiments, and achieved the best results. This indicates that larger update magnitude helps the decoder adapt more effectively to the captioning task without overfitting.

Parameter Efficiency

With only ~2.98M trainable parameters (LoRA matrices in q_proj and v_proj across 28 layers + MLP projector), our model significantly outperforms all baselines. The full decoder has ~1B parameters, meaning we train less than 0.3% of the total model while achieving strong performance.

Architecture Design

Prepending visual tokens before text tokens allows the decoder to attend to image features at every layer via self-attention. The 2-layer GELU MLP projector effectively bridges the 768-dim visual space and 1024-dim language space. Freezing the vision encoder preserves pretrained CLIP visual representations.
