01 — Problem

Object Hallucination in Multimodal LLMs

Large Vision-Language Models (LVLMs) have achieved remarkable performance across vision-language tasks, yet they are prone to a fundamental failure mode: object hallucination — confidently asserting the presence of objects that simply do not exist in the image.

This arises from two sources of bias baked into the model during training:

Bias 01

Statistical Co-occurrence Bias

Models learn that certain objects frequently appear together in training data and generate them regardless of visual evidence.

Bias 02

Language Prior Dominance

Strong language priors cause the model to predict tokens based on likely word sequences rather than actual visual content.

"When asked 'Is there a baseball glove in the image?', a model might answer 'Yes' simply because baseball bats and gloves co-occur frequently in training data — even when no glove is present."

The task is to evaluate LLaVA on the POPE benchmark — a probing dataset of binary yes/no questions about object existence — both with and without VCD, measuring accuracy as the primary metric.

03 — Model

LLaVA-v1.5-7B Architecture

LLaVA (Large Language and Vision Assistant) connects a vision encoder to a language model through a learned projection layer, enabling instruction-following on image-text inputs.

The 1.5 variant uses CLIP ViT-L/14@336px as the visual backbone, a two-layer MLP as the multimodal projector, and Vicuna-7B (LLaMA-2 based) as the language decoder.

Pretrained weights are loaded directly from liuhaotian/llava-v1.5-7b — no finetuning is applied for these experiments.

04 — Method

Visual Contrastive Decoding

VCD is a training-free, zero-shot decoding strategy that suppresses hallucinated tokens by contrasting logits from the original image against logits from a visually degraded counterpart. Tokens that persist even when the visual signal is destroyed are penalized — they are language priors, not genuine visual observations.

Clean image tensor passed to LLaVA
🌫️
Noisy Image
Diffusion noise added at step 500
📊

Forward pass for both inputs in parallel
⚖️

✂️

🎯

Sample next token from adjusted distribution
logits_cd = (1 + α) × logits_clean − α × logits_noisy
cutoff = log(β) + max(logits_clean)
final_logits = logits_cd · 𝟙[logits_clean ≥ cutoff]
α = 1.0 (contrastive weight)  ·  β = 0.1 (plausibility threshold)  ·  noise_step = 500

Noise Generation

The noisy image is generated via the forward diffusion process: a linear noise schedule over 1000 steps, with the noisy variant sampled at step 500 — halfway through the diffusion chain, retaining some structure while heavily corrupting visual detail.

# Forward diffusion at step t x_t = √ᾱ_t · x_0 + √(1−ᾱ_t) · ε ε ~ 𝒩(0, I), t = 500

Adaptive Plausibility Constraint

To prevent the contrastive operation from boosting genuinely unlikely tokens, a plausibility mask filters out tokens whose original log-probability falls below a threshold relative to the most likely token. Only plausible completions are eligible for sampling.

# Mask implausible tokens cutoff = log(β) + logits.max(dim=-1) cd_logits[logits < cutoff] = -inf
05 — Inference Pipeline

Implementation Details

Inference runs zero-shot — no finetuning, no prompt engineering beyond a minimal instruction suffix. Each image-question pair is processed independently through the full pipeline.

Prompt Format

LLaVA-v1 Conversation Template

Uses the llava_v1 conversation template. The question is appended with "Please answer with Yes or No." as a soft directive. Raw model output is post-processed by checking the first word.

Post-processing

Output Normalization

Raw outputs like "Yes, there is a cat" or "No, I don't see any..." are truncated to the leading Yes/No token. Handles casing and punctuation variants robustly.

07 — Observations

Analysis & Discussion

01
VCD Provides Consistent Improvement
VCD improves accuracy from 69.0% to 69.9% — a +0.9 percentage point gain with zero additional training. The improvement confirms that a portion of errors in baseline LLaVA stem from language priors rather than visual misunderstanding.
02
Baseline LLaVA Already Performs Well
At 69.0%, LLaVA without VCD already far exceeds both baselines (49%, 55%), suggesting the model has strong visual grounding. The marginal gain from VCD indicates that the remaining errors are harder to address through decoding alone.
03
Noise Step is a Critical Hyperparameter
A noise step of 500 was used, placing the distorted image at the midpoint of the diffusion chain. Too little noise preserves visual content (weakening the contrast); too much noise destroys all signal. Step 500 provides a practical balance.
04
α Controls Contrast Strength
With α = 1.0, the decoding formula weights the clean logits at 2× and subtracts the noisy logits at 1×. Higher α aggressively suppresses language-prior tokens but risks over-correcting and rejecting valid predictions. Tuning α is task-sensitive.
VCD demonstrates that hallucination in LVLMs is not purely a training problem — it can be meaningfully addressed at inference time by exploiting the contrast between what a model sees and what it would generate blind.
