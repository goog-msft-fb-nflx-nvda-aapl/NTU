Problem 3: ControlNet

Conditional Image Generation with Structural Control · Fill50K Dataset

01 · Overview

Problem Description

The goal of this problem is to implement ControlNet, a neural network architecture that adds spatial conditioning control to large pre-trained text-to-image diffusion models (Stable Diffusion v1.5). Given a control image (a black-and-white circle mask) and a text prompt, the model must generate an image that matches both the structural layout of the control image and the color/content described by the prompt.

The evaluation uses the Fill50K dataset, which contains circle shape images with various color prompts. Generated images are evaluated using:

  • SAM2 mIoU — segmentation shape fidelity between generated and ground truth images
  • CLIP text-image similarity — how well the generated image matches the text prompt
  • Final score = CLIP × mIoU, evaluated on top-10 of 20 generated images

02 · Dataset

Fill50K Dataset

The Fill50K dataset consists of paired source (control) and target images along with text prompts. Each source image contains a black circle on white background, and each target image is the colored version according to the prompt.

03 · Architecture

ControlNet Architecture

ControlNet (Zhang et al., 2023) augments a frozen pre-trained diffusion UNet with a trainable copy of its encoder. The key idea is:

Input: x_noisy (latent), t (timestep), text prompt, hint image (control)

ControlNet (trainable copy of SD encoder): hint → input_hint_block (8 conv layers, 8× downsample) → latent hint x_noisy + latent hint → encoder blocks (12 blocks) → zero_convs → control signals middle block → zero_conv → control signal (middle)

Frozen SD UNet decoder: encoder hidden states + control signals → decoder blocks → x_pred

Zero convolutions: 1×1 conv initialized to zero, ensuring ControlNet starts as an identity and gradually learns to inject control without disrupting SD.

05 · Report Q1

Implementation Details & Difficulties

Implementation Approach

The implementation followed the official ControlNet repository. The key components are:

  • ControlNet module (cldm/cldm.py): A trainable copy of the SD UNet encoder with an additional input_hint_block that downsamples the 512×512 control image to latent space size via 8 strided convolutions. Each encoder block output passes through a zero convolution before being added to the frozen UNet decoder's skip connections.
  • ControlledUNet: The frozen SD UNet decoder is modified to accept the 13 control signals (12 encoder + 1 middle) and add them to the corresponding skip connections.
  • Initialization: The ControlNet encoder weights are copied from the pretrained SD UNet encoder via tool_add_control.py, ensuring the model starts in a stable state.
  • Dataset: The Fill50K dataset loader normalizes source images to [0,1] and target images to [-1,1], consistent with the diffusion model's expected input range.

Difficulties Encountered

Environment compatibility issues: The codebase targets PyTorch ~1.11 and pytorch-lightning 1.4.2, but the available environment runs Python 3.12 with PyTorch 2.6+. This caused several breaking changes: np.Inf removed in NumPy 2.0, torch.load defaulting to weights_only=True, and gradient checkpointing incompatibilities. Each required targeted fixes.
VectorQuantizer import error: The installed taming-transformers package renamed VectorQuantizer2 to VectorQuantizer. Fixed by patching the import in ldm/models/autoencoder.py.
Multi-GPU training: Using DDP across 8 H200 GPUs with pytorch-lightning 1.4.2 required using accelerator='ddp' and gpus=8, which reduced steps per epoch from 12,475 (single GPU) to 780, dramatically speeding up training.

Control Image Experiment: Two Circles

We generated images using ControlNet with control images containing two circles to analyze whether ControlNet can handle multi-object control conditions that differ from the single-circle training distribution.

Discussion

Experiment 1 (two separate circles): Complete failure. The model generates noisy, incoherent textures with no recognizable structure. Two disconnected black regions on white are entirely out-of-distribution — the model cannot reconcile two separate shapes with its single-circle prior learned during training.
Experiment 2 (overlapping circles): Partial success. The model generates two distinct color regions (yellow/green) that roughly follow the merged boundary. Since the overlapping circles appear as one large shape, the model treats them as a single object but partially follows the two-color text prompt.

Possible reasons for failure/weak control:

  • Out-of-distribution input: The model was trained exclusively on single-circle images. Two separate circles are outside the training distribution, causing complete breakdown of the control signal.
  • Control signal ambiguity: The binary mask cannot encode object identity — the model cannot determine how many distinct objects are present from shape alone.
  • Text-control misalignment: The model's learned association between prompt and control was built on single-color, single-circle pairs. Multi-object prompts cannot override the shape prior.
  • Overlapping shapes as workaround: When circles overlap and form a connected region, the model can at least generate a coherent output, suggesting the connectivity of the control mask is critical.
