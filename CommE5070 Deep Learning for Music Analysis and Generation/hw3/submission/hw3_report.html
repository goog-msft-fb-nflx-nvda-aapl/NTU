01 · Overview

Problem & Approach

Given a dataset of 1747 pop piano MIDI files, we train an autoregressive transformer to generate 32-bar symbolic music unconditionally (Task 1), and continue 8-bar prompts for 24 additional bars (Task 2). Music is represented as REMI token sequences and evaluated with pitch entropy (H4) and groove similarity (GS) metrics.

02 · Tokenization

REMI Representation

Each MIDI file is converted into a sequence of REMI events: Bar markers delimit measures, Position tokens encode 1/16th-note grid positions, Tempo Class/Value encode BPM, Note Velocity encodes dynamics (32 bins), Note On encodes pitch (MIDI 0–127), and Note Duration encodes length (60ms–3840ms, 64 bins).

03 · Architecture

GPT-2 Model

We use a GPT-2 causal language model trained to predict the next token in a REMI sequence. The model uses a sliding window of 1024 tokens during both training and inference. Multi-GPU training with DataParallel across 8 H200 GPUs.

06 · Analysis

Discussion

H4 Observation

Config 1 (temp=1.2, k=5) achieves H4=2.6574, the closest to the real data reference of 2.6718. Higher temperature (Config 2) slightly reduces pitch variety, while lower temperature (Config 3) is comparable. The model captures the pitch-class distribution of Pop1K7 well.

GS Observation

Config 3 (temp=0.9, k=3) achieves the highest groove similarity (0.7303), even exceeding real data (0.7182). Lower temperature produces more repetitive rhythmic patterns, which increases GS. Config 1 and Config 2 are more rhythmically varied but still close to the reference.

Best Config

Config 1 (temperature=1.2, top-k=5) is selected as the best overall configuration — it achieves the best H4 score closest to real data, and maintains competitive GS. The 20 generated MIDI/WAV files from this config are submitted as listening samples.

Limitations

The model does not use chord tokens or explicit structural tokens. Longer-range structure (SI metric) was not evaluated. Future work: add chord extraction, use REMI+ or CP Word representation, train with BPE tokenization to reduce sequence length and allow longer context.
