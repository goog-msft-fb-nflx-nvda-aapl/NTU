// 01 — OVERVIEW

What is this project?

This project explores whether a fine-tuned LLM can serve as a reliable cybersecurity knowledge assistant. Using parameter-efficient fine-tuning (LoRA), a Qwen2.5-7B-Instruct model was adapted to answer cybersecurity questions drawn from the MMLU Computer Security benchmark.

The result is a chatbot capable of explaining attack types, defense mechanisms, encryption concepts, and security protocols — deployable on HuggingFace Spaces with zero infrastructure cost.

// 02 — PIPELINE

How it was built

A four-stage pipeline from raw dataset to deployed demo.

01
Data Curation

MMLU Computer Security dataset loaded from HuggingFace. 116 Q&A pairs converted to instruction-following format.

02
Fine-tuning

Qwen2.5-7B-Instruct fine-tuned with LoRA (r=16) using TRL SFTTrainer on NVIDIA H200 GPU.

03
Evaluation

Inference tested on unseen cybersecurity questions. Loss: 3.638 → 2.979 over 5 epochs.

04
Deployment

Gradio chat interface deployed to HuggingFace Spaces. LoRA adapter hosted on HuggingFace Hub.

// 04 — PBL MILESTONES

Project timeline

Four-week project-based learning track from requirements to solution.

WEEK 13 — REQUIREMENTS
Scope & Dataset

Defined project scope: cybersecurity Q&A chatbot. Selected MMLU Computer Security dataset and Qwen2.5-7B as base model.

WEEK 14 — DESIGN
Architecture & Pipeline

Designed LoRA fine-tuning pipeline, instruction prompt format using Qwen chat template, and data preprocessing strategy.

WEEK 15 — POC
Fine-tuning & Inference

Trained LoRA adapter on H200. Verified inference on unseen questions about buffer overflows, SQL injection, and encryption.

WEEK 16 — SOLUTION
Deployment & Demo

Deployed Gradio chat interface on HuggingFace Spaces. Published LoRA adapter to HuggingFace Hub for reproducibility.
