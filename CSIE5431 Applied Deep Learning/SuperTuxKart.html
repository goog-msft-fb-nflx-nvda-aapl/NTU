01 · Task Overview

Goal

Train models that predict a vehicle's future trajectory (waypoints) to enable autonomous driving in SuperTuxKart. Given either ground-truth lane boundaries or a raw RGB image, predict the next 3 waypoints in the vehicle's ego coordinate frame.

Predicted waypoints are fed into a low-level controller that converts them into steering and throttle commands to drive the kart around a track.

🏎️

Part 1a — MLP Planner

Predict waypoints from ground-truth lane boundaries using a multi-layer perceptron. Perfect-vision oracle scenario.

🔭

Part 1b — Transformer Planner

Same task, but using a Perceiver-style cross-attention architecture. Query embeddings attend over track boundary features.

📷

Part 2 — CNN Planner

Predict waypoints directly from raw RGB images. No ground-truth boundaries — the model must learn visual features end-to-end.

02 · Dataset

SuperTuxKart
Drive Dataset

Collected by driving around 4 tracks in SuperTuxKart. Each episode records vehicle state and camera images at every frame. Data is processed into ego-frame coordinates for training.

Field Shape Description
track_left (B, 10, 2) Left lane boundary points in ego BEV frame. 10 points sampled ahead of the vehicle at 2.5m intervals.
track_right (B, 10, 2) Right lane boundary points in ego BEV frame. Mirror of track_left.
waypoints (B, 3, 2) Ground truth future vehicle positions. Target for all models.
waypoints_mask (B, 3) Boolean mask for valid waypoints. Used to ignore padding at episode boundaries.
image (B, 3, 96, 128) Front-facing RGB camera image. Used only by CNN Planner.
Training tracks cornfield_crossing, hacienda, lighthouse, snowmountain (×4 episodes each)
Validation tracks Same 4 tracks, held-out episode _05
Coordinate frame Ego bird's-eye view (BEV), x=lateral, z=longitudinal

05 · Key Findings

What We Learned

01

Lateral error is the hard part

Longitudinal (forward) error converged quickly across all models. Lateral (steering) error was consistently the bottleneck — the model needs to learn left/right geometry from a sparse 20-point representation of the track.

02

Weighted loss fixed the lat problem

Adding a 2× weight on lateral loss in the training objective forced the optimizer to prioritize steering accuracy. Without it, the loss was dominated by longitudinal error and lat stagnated above 0.8.

03

CNN beats oracle models on lateral accuracy

Surprisingly, the image-based CNN (lat=0.225) significantly outperformed the MLP (0.573) and Transformer (0.577) that receive ground-truth lane boundaries. Rich visual context from the image compensates for the lack of explicit geometry.

04

Val loss oscillation ≠ model quality

Both MLP and Transformer showed highly unstable validation loss across epochs (±0.4 swings). Best model checkpointing by val loss was critical — the final epoch weights were often far from optimal.

05

Perceiver cross-attention works out of the box

Using n_waypoints learned query embeddings attending over 20 track points (left+right) is a clean fit for the Perceiver architecture. The model learns which track points are most relevant for each future waypoint.
