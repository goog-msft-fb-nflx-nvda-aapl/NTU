The Shift: From Sparse Points to Neural Maps

ORB-SLAM map (classical)

  • Sparse 3D points + keyframe graph
  • Not renderable — no novel view synthesis
  • No geometry between points
  • Not differentiable — cannot optimize via rendering loss

Neural map (this section)

  • MLP maps (x,y,z) → occupancy + colour
  • Dense — full surface reconstruction
  • Differentiable — optimize both map and pose via backprop
  • Predictive — fills in unobserved regions
Bridge from W04: NeRF showed MLPs can represent scenes offline. iMAP asks: can it work online, in real time, as the camera moves?

iMAP: The MLP as Map

  • Input: RGB-D stream (Azure Kinect)
  • Single MLP F_θ: (x,y,z) → (occupancy, colour)
  • Two parallel processes — same as PTAM spirit
  • Process 1 — Tracking: fix network, optimize pose T at ~10 Hz
  • Process 2 — Mapping: joint optimization of θ + keyframe poses at ~2 Hz
  • Loss: geometric L1 depth + photometric L1 colour
  • Optimizer: ADAM on both θ and {Ti}
  • No prior training — network initialized randomly
The entire scene = 1 MB of network weights. No point cloud. No mesh. Just θ.

iMAP: Active Sampling & Forgetting

Active Pixel Sampling
  • Only 200 pixels/frame rendered per iteration — too expensive otherwise
  • Divide image into 8×8 grid → compute per-cell loss
  • Resample proportional to loss — focus on high-error regions
  • Active sampling beats random: faster convergence, higher completion
Keyframe Selection
  • New KF if proportion of well-explained pixels P < 0.65
  • At each joint opt step: fixed window of W = 5 KFs, sampled by loss
  • Serves as replay buffer — prevents catastrophic forgetting
Critical insight: without KF replay, the MLP forgets previously seen rooms — classic catastrophic forgetting in continual learning

iMAP: Results & Hard Limits

What iMAP achieves
  • Dense watertight reconstruction — fills holes TSDF fusion leaves
  • Whole room in ~1 MB vs TSDF 67 MB at 256³
  • Tracking 10 Hz, mapping 2 Hz — real-time
  • Completion ratio 79% on Replica vs TSDF 75%
Hard limitations
  • Single MLP bottleneck — fixed capacity regardless of scene size
  • Any new observation requires global update of all weights
  • Scales to one room — fails on multi-room apartments
  • Oversmoothed geometry — MLP low-frequency bias
  • Tracking ATE 4.9–5.8 cm on TUM — classical methods do 1–2 cm
Root cause: a global MLP cannot do local updates. Touching one part of the scene potentially corrupts another.

NICE-SLAM's question: how do we get local updates while keeping the neural representation?

NICE-SLAM: Replace MLP with Feature Grids

iMAP: global MLP

  • One network encodes everything
  • Update at position X → affects all positions
  • Scale: one room maximum

NICE-SLAM: feature grid + tiny MLP

  • Voxel grid stores learnable feature vectors
  • Tiny pre-trained MLP decoder converts features → occupancy
  • Update at X → only affects nearby voxels
  • Scale: multi-room apartment, works
Key insight: separate what to store (grid features, optimized online) from how to decode (MLP, pre-trained offline, fixed)

NICE-SLAM: Hierarchical Feature Grid

  • 4 grids total — 3 geometry + 1 colour
  • Coarse (2 m voxels): large structure — walls, floor — enables geometry forecast into unseen areas
  • Mid (32 cm voxels): room-level geometry, main reconstruction grid
  • Fine (16 cm voxels): residual offset on mid-level — high-frequency details
  • Colour grid: jointly optimized with decoder for photometric loss
  • Decoders pre-trained on ConvONet — bring shape priors at each resolution
  • Only feature grids optimized online, decoders fixed
Fine occupancy: op = o¹p + Δo¹p — mid prediction + fine residual

NICE-SLAM: Alternating Optimization

Mapping (per keyframe)
  • Sample M pixels from current frame + selected KFs
  • Stage 1: optimize mid-level grid ω¹ only (geometric loss)
  • Stage 2: jointly optimize mid + fine ω¹, ω²
  • Stage 3: local BA — optimize all grids + colour decoder + K keyframe poses
  • Only update features inside viewing frustum — rest stays fixed
Tracking (per frame)
  • Fix all feature grids, optimize only camera pose {R, t}
  • Variance-weighted depth loss — down-weights uncertain edges
  • Coarse grid provides geometry forecast for unseen areas — enables tracking into new regions

Where Neural SLAM Stands

What neural SLAM gains over classical
  • Dense geometry — full surfaces, not point clouds
  • Hole filling — predicts unobserved regions via learned priors
  • Differentiable — map optimized via rendering loss, enables joint optimization
  • Richer representation for downstream tasks
What neural SLAM still lacks
  • Tracking accuracy: ORB-SLAM2 still wins on TUM (1.0 vs 3.0 cm)
  • No loop closure in NICE-SLAM — acknowledged limitation
  • Needs RGB-D — monocular case much harder
  • Rendering speed: feature grid queries slower than rasterization

The arc so far

  • Classical (ORB-SLAM): sparse, fast, accurate — no density
  • iMAP: dense, differentiable — does not scale
  • NICE-SLAM: dense + scalable — slower rendering
Next: Gaussian SLAM — replaces feature grids with 3D Gaussians. Real-time rendering via rasterization. Closes the W04→W05 arc.
All three neural variants lack what ORB-SLAM has: a mature loop closure system. Open problem as of 2022.

Section 2 — Key Takeaways

  • iMAP: first real-time SLAM with MLP as map — joint optimization of θ and poses via differentiable volume rendering
  • Active sampling + keyframe replay solve real-time and forgetting — but single MLP bottleneck limits scale
  • NICE-SLAM: replaces global MLP with hierarchical feature grids + pre-trained fixed decoders — enables local updates
  • Three geometry levels (coarse 2 m / mid 32 cm / fine 16 cm) + colour grid — each captures different spatial frequency
  • Coarse level enables geometry forecast into unseen areas → more robust tracking
  • NICE-SLAM: 4× faster, 4× fewer FLOPs, far better tracking than iMAP on large scenes

The key question answered

  • iMAP: can NeRF-style MLP do SLAM? Yes, but one room only
  • NICE-SLAM: how to scale? Local feature grids + pretrained decoders
Map representation progression:
sparse points → global MLP → hierarchical grids → 3D Gaussians (next)
