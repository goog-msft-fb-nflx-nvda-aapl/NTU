Why Replace Feature Grids with Gaussians?

Neural SLAM bottleneck

  • Volume rendering = ray marching — N samples × M pixels
  • NICE-SLAM: 0.54 fps · Point-SLAM: 1.33 fps
  • Too slow for real-time tracking feedback loop

3DGS: rasterisation

  • Iterate over Gaussians, not along rays
  • CUDA-native, exploits scene sparsity
  • Rendering: 769 fps on Replica
  • Fully differentiable — gradient flows to Gaussian params and pose T
  • Explicit geometry — easy to insert, prune, warp
3DGS: most like a surfel cloud in structure, but renders with photorealistic quality. Inherits efficiency and locality; gains differentiability.

3D Gaussians as SLAM Primitives

Each Gaussian Gi stores
  • μW — 3D mean position in world coordinates
  • ΣW — covariance matrix (size, orientation, elongation)
  • ci — colour (RGB)
  • αi — opacity
Alpha-blending (splatting)
  • Project 3D Gaussians to 2D via EWA splatting
  • Cp = Σ ciαi Π(1−αj) — front-to-back blend
  • All parameters differentiable → gradient to μ, Σ, c, α and pose T
Adaptive geometry
  • Flat ellipsoids for walls/floors · round for objects · thin elongated for wires
  • Rim-placed Gaussians naturally represent transparent objects
  • No fixed resolution, no mesh topology

System: Tracking · Keyframing · Mapping

Tracking

  • Fix map G, optimize pose T only
  • Photometric loss Epho + geometric Egeo (if depth)
  • ≥50 CUDA gradient steps per frame
  • Affine brightness correction for exposure change

Keyframing

  • New KF if Gaussian IOU drops below threshold or large translation
  • Insert Gaussians: back-project depth (RGBD) or rendered depth (mono)
  • Prune: unseen by ≥3 frames in last 3 KFs → remove

Mapping

  • Window Wk + 2 random past KFs (anti-forgetting)
  • Joint: optimize Gaussian params + KF poses
  • Isotropic regularization Eiso

Two Key Technical Innovations

① Analytic SE(3) Pose Jacobian
  • Original 3DGS requires known poses from SfM — offline only
  • Online SLAM needs gradient of rendering loss w.r.t. camera pose T
  • First analytic Jacobian on SE(3) Lie group for EWA splatting
  • Minimal 6-DoF parametrization — no redundant computation
  • Integrated into CUDA rasterizer — no autograd overhead
  • ≥50 gradient steps/frame feasible at interactive speed
This was unexplored before this paper — opening 3DGS to all online pose-optimization applications beyond SLAM
② Isotropic Regularization Eiso
  • Without depth, Gaussians elongate along viewing ray — photometric loss cannot observe this
  • Elongation causes artifacts in novel views + corrupts tracking
  • Eiso = ‖si − s̃i·1‖₁ penalizes deviation from spherical shape

Surprise Result: Wide Convergence Basin

  • 3DGS maps empirically show a notably larger convergence basin for pose optimization
  • Test: train map from 9 views; localize from 0.2–1.2 m away (wider than training baseline)
  • 3DGS w/ depth: 82% success · Hash Grid SDF: 14% · MLP SDF: 33%
  • Even monocular 3DGS (no depth): 79%
Why? Anisotropic Gaussians form a smooth gradient field — no hash collisions, no positional-encoding aliasing. Gradient persists over larger displacements.
  • Practical: robust tracking after fast motion or frame drops
  • Useful for map-based relocalization from wider viewpoints

Limitations & Closing the W04→W05 Arc

Current limitations
  • No loop closure — drift inevitable over long trajectories
  • Small-scale only — single rooms; large scenes degrade
  • 3 fps end-to-end — not hard real-time
  • Monocular scale ambiguity remains
Authors note: loop closure should be easier for Gaussians than voxel grids — Gaussians can be warped like surfel clouds (cf. ElasticFusion). Key open direction.
The frontier: unify Gaussian SLAM's rendering quality with ORB-SLAM's pose accuracy and loop closure. No system achieves both today.

The W04→W05 arc complete

  • W04: Occ / NeRF / 3DGS as offline reconstruction
  • W05: same representations, now online SLAM

Occ/TSDF grid → KinectFusion, Vox-Fusion

Same pipeline pattern across all: tracking alternates with mapping. The map representation is the design variable — it determines what the system can perceive, render, and correct.

Section 3 — Key Takeaways

  • Gaussian SLAM: first SLAM using 3DGS as the only representation — unified tracking, mapping, rendering
  • Rasterisation replaces ray marching → 769 fps rendering, orders of magnitude faster than neural methods
  • Analytic SE(3) Jacobian enables gradient-based pose tracking against Gaussians — novel contribution
  • Isotropic regularization prevents elongation artifacts — essential for incremental monocular reconstruction
  • Covisibility IOU keyframing + random past KF replay handle memory and forgetting
  • RGB-D tracking 1.47 cm avg — competitive with ORB-SLAM2 · Mono 3.96 cm — best no-prior monocular method

What 3DGS uniquely brings

  • Near-photorealistic maps at interactive rates
  • Wide convergence basin → robust relocalization
  • Handles transparency, wires, complex materials
  • Compact: ~4 MB per room
