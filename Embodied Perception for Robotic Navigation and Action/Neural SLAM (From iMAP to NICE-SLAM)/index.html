Section 01

Background You Need

Before diving in, make sure you have a mental model of these four concepts. Each card is a 60-second primer.

🗺️

Classical SLAM

Simultaneous Localization And Mapping. A robot asks: "where am I, and what does the world look like?" — at the same time, from scratch. Traditional methods use voxels, point clouds, or occupancy grids.

🧠

Neural Implicit Representation

Instead of storing a grid of values, we train a neural network f(x,y,z) → (density, color). The network is the map. Compact, differentiable, and smooth.

📷

RGB-D Camera

A camera that captures both color (RGB) and depth (D) per pixel. Used in both iMAP and NICE-SLAM as input. Think: Intel RealSense, Azure Kinect.

🌊

Volume Rendering

Cast rays from the camera. Sample points along each ray. Ask the neural network what density and color each point has. Integrate to get the final pixel color — differentiable end-to-end.

Key intuition: A NeRF-style network is essentially a compressed, queryable 3D map. SLAM systems learned to use this as their scene representation — and optimize it in real time.

Context

The Road to Neural SLAM

2020 — NeRF

Neural Radiance Fields

Mildenhall et al. show a simple MLP can represent an entire 3D scene for novel view synthesis. No SLAM — takes hours to train on a static scene.

2021 — iMAP (ICCV)

First Neural Implicit SLAM

Sucar et al. replace the voxel map with a single MLP. Train it live as the robot moves. Tracks at 10 Hz. Small rooms only — catastrophic forgetting at scale.

2022 — NICE-SLAM (CVPR)

Hierarchical Grid + Neural Decoders

Zhu, Peng et al. replace the single MLP with multi-scale feature grids + lightweight decoders. Local updates. Scales to entire apartments. Competitive tracking.

2022–2024 — The Wave

ESLAM, Co-SLAM, Point-SLAM, MonoGS…

NICE-SLAM sparked a wave of follow-ups: hash grids, Gaussian splatting, monocular depth — neural SLAM became a core research frontier.

Section 02 · ICCV 2021

iMAP — The Pioneer

iMAP is the first system to use a single MLP as the sole scene representation in a real-time RGB-D SLAM pipeline. No point clouds, no voxels — just one network.

How It Works

A fully-connected MLP with 4 hidden layers takes a 3D point (x,y,z) and returns (occupancy, color). Volume rendering composites these samples along camera rays into predicted RGB and depth images.

The system alternates between two processes:

  • Tracking — freeze the map, optimize camera pose
  • Mapping — freeze the pose, update the MLP weights
f_θ(x, y, z) → (σ, c)

Keyframes are stored and replayed to fight forgetting. Dynamic pixel sampling focuses compute on high-information regions.

The Limitations

Catastrophic Forgetting
A single global MLP must represent the entire scene. Updating one region degrades others.

Over-smoothing
The network's natural bias toward low-frequency functions produces blurry, detail-free geometry.

No Scalability
Mapping rate: ~2 Hz. Works for single small rooms. Falls apart for large or multi-room scenes.

# iMAP: the core idea in pseudocode class iMAPSystem: def __init__(self): # The ENTIRE map lives in one MLP self.map_network = MLP( input_dim=3, # (x, y, z) hidden_dim=256, output_dim=4 # (occupancy, R, G, B) ) self.keyframes = [] def tracking(self, frame): # Freeze network, optimize camera pose T T = optimize_pose(frame, self.map_network) return T def mapping(self, frame, T): # Freeze pose, update ALL weights globally # ⚠ Updating room B corrupts room A! gradient_step(self.map_network, frame, T) replay_keyframes(self.keyframes) # fight forgetting
Section 03 · CVPR 2022

NICE-SLAM — The Fix

NICE-SLAM's core insight: decouple geometry storage from geometry decoding. Store features in a spatial grid (local), decode geometry with a small pre-trained network (global).

The Hierarchical Representation

Three scales of feature grids cover the scene:

  • Coarse grid — low-res, for fast camera pose recovery
  • Mid grid — medium detail geometry
  • Fine grid — high-res fine geometry
  • Color grid — separate appearance features

Each grid cell stores a small learnable feature vector. A shared, pre-trained decoder MLP reads features and outputs occupancy/color.

F_grid(x) → feat → Decoder_θ(feat) → (σ, c)

Why This Solves Everything

Local updates, no forgetting
Updating a region only touches nearby grid cells. Other regions are completely untouched.

Sharp geometry
Pre-trained priors on the decoder know what "flat wall" or "sharp edge" should look like before seeing a single frame.

Scales to apartments
Just make the grid bigger. Storage grows with scene size, but compute stays local.

Property iMAP NICE-SLAM
Scene Representation Single global MLP Multi-scale feature grids + decoders
Update Locality Global (all weights) Local (nearby grid cells only)
Catastrophic Forgetting Severe — needs keyframe replay Minimal — grids are independent
Scene Scale Single small room Multi-room apartments
Geometry Detail Over-smoothed, blurry Sharp edges via geometric priors
Geometric Priors None (train from scratch) Pre-trained on ShapeNet
Tracking Speed ~10 Hz ~10 Hz
Datasets Evaluated 2 5 (Replica, ScanNet, TUM, Co-fusion…)

Section 05

Check Your Understanding

Test yourself on the key concepts. Click an option to see if you're right.

1. What is the primary advantage of NICE-SLAM's hierarchical feature grid over iMAP's single MLP?
✓ Correct! Because features are stored spatially in a grid, gradient updates only affect nearby voxels. iMAP's global MLP weights are shared across the whole scene — updating any region modifies the entire representation.
2. In volume rendering, what does a point with high density (σ) contribute to the final pixel color?
✓ Correct! The volume rendering integral weights each sample's color by its transmittance × density. High-density points (surfaces) contribute strongly; empty space contributes almost nothing.
3. NICE-SLAM's geometric decoder is "pre-trained." What does this mean and why does it help?
✓ Correct! Pre-training on ShapeNet teaches the decoder what surfaces look like. When SLAM begins, the decoder already understands geometry — so even with limited observations it produces clean, detailed reconstructions instead of blurry blobs.
4. What is "catastrophic forgetting" in the context of iMAP?
✓ Correct! This is the classic continual learning problem. A neural network trained sequentially on different data regions "forgets" earlier data. iMAP mitigates this with keyframe replay, but it's still a fundamental limitation of the single-MLP design.
