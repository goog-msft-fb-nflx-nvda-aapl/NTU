Problem Description

The path tracking task requires a simulated kinematic vehicle to follow a cubic-spline generated reference path. The agent controls a single continuous action — angular velocity — and is rewarded for staying close to the path and maintaining correct heading orientation.

Action & Termination

Action Space

Single continuous action: angular velocity ∈ [−1, 1]. Values outside range are clipped automatically.

Episode Termination

Episode ends at 100% path progress or after reaching 400 timesteps (max steps).

Reward Function

r = 0.8 r_d + 0.2 r_y + r_p // total reward (fixed by environment)
r_d = exp(−0.1 ‖p − p'‖²) // distance reward: penalizes deviation from path
r_y = exp(−0.1 ‖θ − θ'_normal‖²) // yaw reward: penalizes misalignment with path tangent
r_p = progress bonus // reward for advancing along path

Policy Network (PolicyNet)

A two-layer MLP maps the 14-dimensional observation to a feature vector, which is then passed through a diagonal Gaussian distribution head to produce stochastic actions.

class PolicyNet(nn.Module):
    def __init__(self, s_dim, a_dim, std=0.5):
        self.main = nn.Sequential(
            init_(nn.Linear(s_dim, 64)), nn.ReLU(),
            init_(nn.Linear(64, 64)),  nn.ReLU(),
        )
        self.dist = DiagGaussian(64, a_dim, std)   # fixed std=0.5

    def evaluate(self, state, action):
        dist = self.dist(self.main(state))
        return dist.log_probs(action), dist.entropy()

Value Network (ValueNet)

A symmetric two-layer MLP estimates the state value V(s), used for computing advantages during training.

class ValueNet(nn.Module):
    def __init__(self, s_dim):
        self.main = nn.Sequential(
            init_(nn.Linear(s_dim, 64)), nn.ReLU(),
            init_(nn.Linear(64, 64)),  nn.ReLU(),
            init_(nn.Linear(64, 1)),
        )

    def forward(self, state):
        return self.main(state)[:, 0]   # scalar value per state

EnvRunner — Parallel Data Collection

The EnvRunner collects n-step trajectories from 16 parallel environments simultaneously. After each rollout, Generalized Advantage Estimation (GAE) computes returns and advantages.

PPO Training Loop

The clipped surrogate objective prevents destructively large policy updates. Advantages are normalized per mini-batch. Both actor and critic are updated with gradient clipping (max norm = 0.5).

# Per mini-batch advantage normalization
sample_advs = (sample_advs - sample_advs.mean()) / (sample_advs.std() + 1e-8)

# Clipped surrogate loss
ratio   = torch.exp(sample_a_logps - sample_old_a_logps)
pg_loss1 = -sample_advs * ratio
pg_loss2 = -sample_advs * torch.clamp(ratio, 1-clip_val, 1+clip_val)
pg_loss  = torch.max(pg_loss1, pg_loss2).mean()

# Clipped value loss
v_pred_clip = old_values + torch.clamp(values - old_values, -clip_val, clip_val)
v_loss = torch.max((returns-values).pow(2), (returns-v_pred_clip).pow(2)).mean()

Observations & Analysis

  • Rapid early learning (iters 0–1,000): Mean return rose sharply from ~39 to ~128, indicating that the reward signal (distance + yaw alignment) provides clear gradient information from the start. The two-timestep pose history likely helps the agent infer velocity.
  • Saturation plateau (iters 8,000–30,000): Return stabilized in the 220–250 range after roughly 8,000 iterations, suggesting the policy converged to a near-optimal strategy for the given network capacity. Further gains required architectural changes.
  • Variance reduction over time: Standard deviation of returns decreased from ~40 at early training to ~18–20 at convergence, reflecting a more consistent and reliable policy rather than erratic behavior.
  • Occasional dips: Short-lived drops (e.g., iter 9,200: 179.3, iter 4,100: 169.5) are characteristic of PPO's stochastic mini-batch sampling. The policy recovers quickly, validating the clipping mechanism.
  • Mini-batch advantage normalization: Normalizing advantages per mini-batch (rather than globally) improved training stability and allowed a higher epoch count (8) without divergence.
  • ReLU vs Tanh: ReLU activations were used in the final trained model. For this bounded observation space (x,y ∈ [0,1], yaw ∈ [0,2π]), ReLU performed comparably to Tanh without the saturation risk at extreme inputs.

Future Improvements

01

Larger Network Capacity

The current 64-unit two-layer MLP may bottleneck performance. Scaling to 128 or 256 units, or adding a third layer, could allow the policy to learn more nuanced path-following strategies, particularly for sharp turns.

02

Adaptive Action Standard Deviation

The fixed action std (σ=0.5) is a strong prior. Learning σ jointly with the policy (via a log-std parameter) would allow the agent to automatically reduce exploration as training progresses, potentially improving final performance.

03

Entropy Regularization

Adding an entropy bonus to the policy loss (ent_coef > 0) would explicitly encourage exploration and prevent premature convergence to a suboptimal deterministic policy, especially beneficial in early training.

04

Longer Rollouts / More Environments

Increasing n_step beyond 128 or n_env beyond 16 would provide better return estimates and more diverse training data per iteration. With 8× H200 GPUs available, a distributed PPO implementation could scale this significantly.

05

Curriculum Learning

Starting training on simpler, lower-curvature paths and gradually increasing difficulty could accelerate learning and lead to better generalization across the randomized spline paths generated at each episode reset.
