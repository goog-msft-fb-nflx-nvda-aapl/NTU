01 — Problem

Navigate. Survive. Capture.

The agent operates in Proly, a 2.5D racing game built in Unity. The map contains 10 sequentially ordered flags. The agent must capture them all as fast as possible while staying alive — falling into water resets health to zero.

The challenge: the agent receives only a compact observation vector and must learn to move, navigate obstacles, and avoid cliffs without any hand-crafted movement logic.

Actions are continuous 2D accelerations (Ax, Ay) ∈ [−1, 1]², giving full directional freedom at every timestep.

target_position
Relative vector to next flag
agent_health
Current health (0 = dead)
last_checkpoint_index
Index of last flag captured
terrain_grid
5×5 local terrain map
0 normal · −1 water · 1 obstacle

02 — Reward Design

Four signals.
One objective.

🚩
Flag Capture
+10.0 × Δcheckpoint_index
The dominant signal. A large reward fires the moment a new flag is captured, directly aligning the agent with the evaluation metric.
📍
Distance Shaping
+0.1 × (prev_dist − curr_dist)
Dense per-step reward for closing in on the next target. Penalizes drifting away. Accelerates early learning before the first flag is reached.
❤️
Survival
−10.0 if health ≤ 0
+0.01 per alive step
A large death penalty discourages cliff falls. The small alive bonus discourages unnecessary idling and rewards continuous movement.
🌊
Terrain Awareness
−0.05 per adjacent water tile
Penalizes standing next to water in the 4-connected neighborhood. Teaches edge avoidance without any hard-coded rules.

03 — Training

PPO on three maps,
in parallel.

Three independent models are trained — one per map — using a MAP_ID environment variable to route each run to its own checkpoint file. The best-generalizing model is selected as the final agent.

01
Game loop → observation

MLGame3D bridge delivers raw observation dict each frame. Flattened vector fed to PPO policy.

02
Reward computation

RewardManager compares current and previous observation to compute the composite reward signal.

03
Rollout buffer fill

Transitions accumulate in the SB3 rollout buffer. When full (2048 steps), returns and advantages are computed.

04
Policy update

PPO gradient update runs at episode reset. Model checkpoint saved after every update.

05
Resume-safe checkpointing

Training can be interrupted and resumed at any time — model reloads from the last saved zip automatically.
